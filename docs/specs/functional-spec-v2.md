# 약 조심 기능 명세서 v2

## 1. 문서 목적

- 이 문서는 현재 개발 기준이 되는 최신 기능 명세서다.
- 기존 [`plan.md`](../../plan.md)는 초기 기획안으로 보존하고, 실제 MVP 개발은 본 문서를 우선 기준으로 한다.
- 이번 버전은 회원 기능을 제외하고 비회원 전용 MVP를 기준으로 정리한다.

## 2. 서비스 한줄 정의

사용자가 약, 음식, 영양제 조합을 입력하면 위험 조합 여부를 `금기`, `주의`, `확인 정보 없음`으로 안내하고, 필요한 경우 시간 간격 또는 상담 필요 정보를 함께 제공하는 복약 안전 안내 서비스

## 3. 이번 버전의 핵심 방향

- 로그인 없이 바로 사용할 수 있는 비회원 전용 MVP로 개발한다.
- 홈 화면은 `약 검색`, `분석`, `설정` 중심으로 단순화한다.
- 처방전 촬영은 독립 메인 기능이 아니라 `약 검색` 또는 `분석` 진입 후 사용할 수 있는 보조 입력 방식으로 둔다.
- 상단 검색창과 `약 검색` 액션의 중복은 제거한다.
- 위험 등급은 `금기 / 주의 / 확인 정보 없음` 3단계로 단순화한다.
- `시간 간격 필요`, `상담 권장`은 별도 등급이 아니라 `주의` 결과의 상세 안내로 제공한다.
- 저장 기능, 마이페이지, 최근 분석 기록은 이번 MVP 범위에서 제외한다.
- 결과 공유는 유지하며, 공유물에는 분석 항목 이름을 모두 포함한다.

## 4. 사용자 범위

### 4-1. 사용자 정책

- 현재 서비스는 비회원만 지원한다.
- 로그인, 회원가입, 마이페이지는 이번 MVP 범위에 포함하지 않는다.
- 분석 결과는 계정 저장 없이 현재 세션 기준으로만 확인한다.

### 4-2. 서비스 원칙

- 의료 진단, 처방, 복용 중단 지시를 하지 않는다.
- 고위험 조합은 반드시 전문가 상담 안내 문구를 노출한다.
- 데이터에 없는 조합은 `안전함`이 아니라 `확인 정보 없음`으로 표시한다.
- AI는 위험도 판단자가 아니라 설명 보조 역할만 수행한다.

## 5. 전체 사용자 흐름

```text
서비스 접속
→ 서비스 안내 및 동의
→ 기본 정보 입력
→ "바로 분석하시겠습니까?" 확인
→ 예: 분석 화면 진입
→ 아니오: 홈 화면 진입
→ 약 검색 또는 분석 시작
→ 약/음식/영양제 선택
→ 조합 분석
→ 결과 확인
→ 이미지/PDF 저장 또는 공유
```

## 6. 온보딩 및 초기 정보 입력

### 6-1. 첫 진입 흐름

1. 사용자는 서비스 접속 후 안내 문구와 면책 문구를 확인한다.
2. 동의 후 기본 정보 입력 화면으로 이동한다.
3. 기본 정보 입력 후 `바로 분석하시겠습니까?` 안내를 보여준다.
4. 사용자가 `예`를 선택하면 분석 화면으로 이동한다.
5. 사용자가 `둘러보겠습니다`를 선택하면 홈 화면으로 이동한다.

### 6-2. 초기 입력 정보

| 항목 | 입력 방식 | 비고 |
| --- | --- | --- |
| 생년월일 | 날짜 입력 | 고령층 모드 자동 판단 기준 |
| 성별 | 단일 선택 | male, female, other, unknown |
| 주요 만성질환 | 다중 선택 | 고혈압, 당뇨, 갑상선, 신장질환, 간질환 등 |
| 임신 여부 | 단일 선택 | 해당 사용자만 입력 |

### 6-3. 고령층 모드 정책

- 고령층 모드는 설정 화면에 유지한다.
- 생년월일 입력값을 기준으로 초기 자동 ON/OFF를 적용한다.
- 사용자는 설정 화면에서 수동으로 다시 ON/OFF 할 수 있다.
- 자동 판단과 수동 설정이 모두 가능해야 한다.

## 7. 홈 화면 정의

### 7-1. 홈 화면 목적

- 서비스 핵심 진입점을 단순하게 제공한다.
- 사용자가 `약 검색` 또는 `분석`으로 바로 이동할 수 있게 한다.
- 랜딩 성격의 소개 디자인을 포함한다.

### 7-2. 홈 화면 구성

- 랜딩 섹션
- 주요 액션 카드: `약 검색`, `분석`
- 하단 메뉴바: `홈 / 검색 / 분석 / 설정`

### 7-3. 홈 화면 규칙

- 기존 상단 검색창은 제거한다.
- `처방전 촬영`은 홈 메인 카드로 직접 노출하지 않는다.
- `약 검색`과 `분석` 내부에서 카메라 촬영 옵션을 제공한다.
- 랜딩 디자인은 홈 화면을 의미하며, 첫 인상과 핵심 가치 전달을 담당한다.

## 8. 주요 화면 정의

### 8-1. 약 검색 화면

- 기본 진입 방식은 텍스트 검색이다.
- 보조 방식으로 카메라 촬영 또는 처방전 사진 인식 기능을 제공한다.
- OCR 수행 후 다음과 같은 안내를 제공할 수 있다.
  - `~로 인식되었습니다`
  - `이 약으로 검색하시겠습니까?`
- 사용자는 검색 결과에서 약을 선택해 분석 목록에 추가한다.

### 8-2. 분석 화면

- 홈의 `분석` 버튼을 누르면 바로 조합 입력 화면으로 이동한다.
- 이 화면에서 약, 음식, 영양제를 추가하고 분석을 실행한다.
- 음식·영양제는 자유 텍스트보다 `검색 후 등록` 방식으로 제공한다.
- 처방전 촬영은 이 화면에서도 보조 입력으로 접근 가능하다.

### 8-3. 음식/영양제 추가 방식

- MVP에서는 대표 목록 또는 검색 기반 선택 방식을 사용한다.
- 자유 텍스트 입력은 이번 MVP 필수 기능에서 제외한다.
- 매핑 실패 처리나 LLM 기반 후보 추천은 후속 확장 항목으로 둔다.

### 8-4. 설정 화면

- 고령층 모드 토글
- 도움말 또는 안내 영역
- 데이터 관리 관련 항목은 후속 논의 대상으로 둔다.

### 8-5. 결과 화면

- 전체 위험 요약
- 조합별 결과 카드
- 상세 설명
- 시간 간격 또는 상담 권장 안내
- 이미지/PDF 저장 및 공유

## 9. 분석 규칙

### 9-1. 분석 대상

- 약
- 음식
- 영양제

### 9-2. 분석 가능 조건

- 조합 분석은 최소 2개 이상 항목이 선택되어야 한다.
- 음식만 단독 선택한 경우 분석 불가로 처리한다.
- 최소/최대 분석 가능 항목 수는 백엔드 API 제약을 우선 기준으로 연동한다.
- 프론트엔드는 API에서 제공하는 min/max 제약을 인지하고 입력 단계에서 안내한다.

### 9-3. 분석 방식

- 제품명은 성분명 기준으로 매핑 후 분석한다.
- 상호작용 판단은 룰 기반 데이터로 수행한다.
- AI는 결과를 쉬운 문장으로 설명하는 데만 사용한다.

## 10. 위험도 정책

### 10-1. 위험 등급

| 등급 | 의미 | 사용자 표현 |
| --- | --- | --- |
| 금기 | 함께 복용하면 위험도가 높아 전문가 확인이 필요한 조합 | 함께 복용 전 반드시 의사·약사와 상담하세요 |
| 주의 | 복용 시간 조정, 상담, 섭취 주의 등이 필요한 조합 | 복용 방법과 섭취 시점을 확인하세요 |
| 확인 정보 없음 | 확인된 상호작용 데이터가 없는 조합 | 안전함을 의미하지 않으며 확인된 정보가 없습니다 |

### 10-2. 상세 안내 정책

- `시간 간격 필요`는 `주의` 결과의 하위 안내로 표시한다.
- `상담 권장`은 `주의` 또는 `금기` 결과에서 추가 메시지로 표시한다.
- 필요 시 결과 카드 또는 모달에서 특이사항을 강조한다.

## 11. 음식/영양제 데이터 정책

### 11-1. 음식 입력 정책

- MVP에서는 대표 음식군 중심으로 제공한다.
- 예: 자몽류, 유제품, 카페인, 알코올
- 음식 `food_group` 세부 기준은 별도 데이터 설계 논의를 따른다.

### 11-2. 영양제 입력 정책

- 제품명이 아니라 성분 또는 대표 항목 기준 검색을 우선한다.
- 주성분 자동 추론이나 외부 추출은 후속 확장으로 둔다.

## 12. 결과 및 공유 정책

### 12-1. 결과 화면 원칙

- 가장 높은 위험도 결과를 상단에서 먼저 보여준다.
- 결과는 `금기`, `주의`, `확인 정보 없음` 기준으로 정렬 또는 필터링할 수 있다.
- 각 결과에는 이유, 쉬운 설명, 행동 가이드를 함께 제공한다.

### 12-2. 공유 정책

- 결과는 이미지 또는 PDF로 저장 가능해야 한다.
- 공유물에는 분석에 사용된 항목 이름을 모두 포함한다.
- 약물명, 음식명, 영양제명은 개수만 보여주지 말고 실제 이름을 노출한다.
